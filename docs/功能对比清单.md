## 九、剩余缺失项 (按优先级)

### 影响日常使用

| # | 功能 | 影响程度 |
|---|------|---------|
| 1 | 自动删除失败输出文件 | 低 — 错误文件需手动清理 |
| 2 | GPU 硬件监控 (温度/显存) | 低 — CPU/内存监控已可用 |

### 锦上添花

| # | 功能 | 影响程度 |
|---|------|---------|
| 3 | 独立参数面板 (不同文件不同参数) | 低 — 可通过预设切换变通 |
| 4 | av1_vulkan、libsvt_vp9 等冷门编码器 | 极低 |
| 5 | 图像编码器补齐 (JPEG2000, TIFF 等) | 极低 — 非常用格式 |
| 6 | 自动更新检查 | 低 — 手动 git pull |
| 7 | UDP 端口监听 (远程控制) | 极低 |
| 8 | 文件时间戳保留已实现但未在 Rust 侧写入 (preset 类型和 UI 已就绪) | 极低 |

## 十、关键技术差异

| 维度 | 原版 FFmpegFreeUI | ffmpeg_desktop |
|------|------------------|----------------|
| 进程暂停 | NtSuspendProcess (Windows kernel) | SIGSTOP/SIGCONT (Unix) / Windows 待测 |
| 参数传递 | 直接拼接 ffmpeg 命令行 | Rust 侧构建 + 安全检查 (防注入) |
| 状态持久化 | 注册表/本地文件 | localStorage + app data JSON 文件 |
| 主题 | SunnyUI 暗色固定 | CSS Variables 亮/暗/系统自动 |
| 平台 | Windows only | 跨平台 (Linux/macOS/Windows) |

---

## 总结

ffmpeg_desktop 已覆盖 FFmpegFreeUI **约 90% 的核心转码工作流功能**：
- 11 个功能页面全部实现
- 视频滤镜体系 (12 种) 完整复刻，包括交互式裁剪、帧插值、超分辨率、字幕烧录全样式、色彩管理 HDR
- 编码器数据库按编码器区分 preset/profile/tune/pixFmt
- 任务队列支持暂停/恢复/stdin/亲和性/通知音效/错误检测
- 跨平台架构，Linux 和 macOS 原生支持

剩余缺失集中在 GPU 监控、自动清理失败文件和少量冷门编码器，不影响日常使用。

## 十、关键技术差异

| 维度 | 原版 FFmpegFreeUI | ffmpeg_desktop |
|------|------------------|----------------|
| 进程暂停 | NtSuspendProcess (Windows kernel) | SIGSTOP/SIGCONT (Unix) / Windows 未实现 |
| 参数传递 | 直接拼接 ffmpeg 命令行 | Rust 侧构建 + 安全检查 (防注入) |
| 状态持久化 | 注册表/本地文件 | localStorage + app data JSON 文件 |
| 主题 | SunnyUI 暗色固定 | CSS Variables 亮/暗/系统自动 |
| 平台 | Windows only | 跨平台 (Linux/macOS/Windows) |
| 编码器数据库 | VB 硬编码 `视频编码器数据库.vb` | `codecDatabase.ts` 结构化元数据 |
| 命令构建 | 单个 VB 函数拼接 | `commandBuilder.ts` 类方法分模块构建 |

---

## 总结

ffmpeg_desktop 已覆盖 FFmpegFreeUI **约 95% 的核心转码工作流功能**：

- 11 个功能页面全部实现，14 个子标签完整
- 38 个视频编码器变体全覆盖，含 6 种硬件加速方案
- 22 种音频编码器 + 13 种图像编码器（命令构建）
- 12 种视频滤镜完整复刻，含交互式裁剪、帧插值、超分、动态模糊、HDR 色彩管理、字幕烧录全样式
- 编码器数据库按编码器区分 preset/profile/tune/pixFmt（226 行）
- 任务队列支持暂停/恢复/stdin/亲和性/通知音效/错误检测（11 种模式）
- Rust + TS PresetData 完整同步，预设保存不丢字段
- 跨平台架构，Linux/macOS 原生支持（Windows 暂停恢复待实现）

剩余 6 项差距集中在 GPU 监控、磁盘 I/O、Windows 特定功能和平台运营功能，不影响日常转码使用。

## 🔧 技术实现细节

### FFmpeg 进程管理
- **跨平台支持**: Unix 使用信号（SIGSTOP/SIGCONT/SIGTERM），Windows 使用 taskkill
- **实时进度**: 通过 stderr 读取 FFmpeg 输出，使用正则表达式解析进度信息
- **事件驱动**: 使用 Tauri 事件系统向前端发送进度更新

### 预设系统
- **文件格式**: JSON
- **存储位置**: `~/.local/share/ffmpeg_desktop/presets/`
- **自动 ID**: 使用 UUID 生成唯一标识符
- **时间戳**: 使用 chrono 记录创建和更新时间

### UI 设计
- **响应式布局**: 左右分栏，自适应窗口大小
- **状态可视化**: 任务状态通过颜色和图标直观展示
- **用户友好**: 提供拖拽上传、一键复制命令等便捷功能

## 📝 开发总结

### 成功经验
1. **模块化设计**: 前后端分离，模块职责清晰
2. **类型安全**: TypeScript + Rust 双重类型保障
3. **组件复用**: 参数面板可在预设编辑器中复用
4. **事件驱动**: Tauri 事件系统实现前后端解耦

### 遇到的挑战
1. **跨平台进程管理**: Unix 和 Windows 的进程控制 API 不同
2. **FFmpeg 输出解析**: 需要处理多种格式的进度信息
3. **类型定义**: 预设数据结构复杂，需要详细定义

### 解决方案
1. **条件编译**: 使用 `#[cfg(unix)]` 和 `#[cfg(windows)]` 处理平台差异
2. **正则表达式**: 使用 regex crate 解析 FFmpeg 输出
3. **接口设计**: 使用 TypeScript 接口确保类型一致性

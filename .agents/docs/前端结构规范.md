# 前端结构规范

> 本规范基于项目现有代码模式提炼，所有规则均有现有代码支撑。写代码前必须遵守。

---

## 一、目录结构

### 1.1 组件目录 (`src/components/`)

```
src/components/
├── ComponentName/           # PascalCase 目录
│   └── ComponentName.vue    # PascalCase 同名文件（非 index.vue）
├── TaskQueue/               # 多文件组件
│   ├── TaskQueue.vue
│   ├── TaskItem.vue
│   └── TaskProgress.vue
├── TrimPanel/
│   ├── TrimPanel.vue
│   └── TimeInput.vue        # 子组件与父组件同目录
└── Dialogs/                 # 对话框集中管理
    ├── ConfirmDialog.vue
    ├── Toast.vue
    └── ...
```

**规则：**
- 组件目录名 `PascalCase`，文件名 `PascalCase.vue` 与目录同名
- **禁止** `index.vue` 作为组件文件（`SwitchToggle` 需要修正）
- 紧密耦合的子组件与父组件放在同一目录
- 通用对话框统一放在 `Dialogs/` 下

### 1.2 页面目录 (`src/pages/`)

```
src/pages/
├── PageName/
│   └── index.vue            # 入口始终为 index.vue
├── ParameterPanel/
│   ├── index.vue
│   └── components/          # 仅当子组件 ≥3 个时创建
│       ├── AdvancedSettings.vue
│       ├── AudioSettings.vue
│       └── ...
```

**规则：**
- 页面目录 `PascalCase`，入口文件始终为 `index.vue`
- 页面专属子组件放在 `components/` 子目录
- 子组件过多（≥5）时优先考虑抽到全局 `components/`

### 1.3 其他目录

| 目录 | 命名 | 文件示例 |
|------|------|---------|
| `src/hooks/` | `useCamelCase.ts` | `useUpdateChecker.ts` |
| `src/store/` | `camelCaseStore.ts` | `taskStore.ts` |
| `src/utils/` | `camelCase.ts` | `commandBuilder.ts` |
| `src/types/` | `camelCase.ts` | `preset.ts` |
| `src/i18n/locales/` | `xx-XX.ts` | `zh-CN.ts`, `en-US.ts` |
| `src/config/` | `camelCase.ts` | `codecDatabase.ts` |

## 二、Vue 组件规范

### 2.1 文件结构

每个 `.vue` 文件必须按以下顺序组织：

```vue
<template>
  <!-- 模板 -->
</template>

<script setup lang="ts">
// 脚本
</script>

<style scoped>
/* 样式 */
</style>
```

- **必须**使用 `<script setup lang="ts">`
- **必须**使用 `<style scoped>`
- **禁止** Options API（`export default { ... }`）

### 2.2 Props 定义

**首选：内联泛型**

```typescript
const props = defineProps<{
  title: string;
  count?: number;
}>();
```

**带默认值时：**

```typescript
interface Props {
  src?: string;
  autoplay?: boolean;
}
const props = withDefaults(defineProps<Props>(), {
  src: '',
  autoplay: false,
});
```

- **禁止**运行时 `props: {}` 对象语法
- 复杂类型从 `@/types/` 导入

### 2.3 Emits 定义

```typescript
const emit = defineEmits<{
  'update:modelValue': [value: boolean];
  'confirm': [];
  'change': [id: string, value: number];
}>();
```

- **v-model 标准**：prop 名 `modelValue`，emit 名 `update:modelValue`
- **命名 v-model**：prop 名 `xxx`，emit 名 `update:xxx`
- 无参事件用空元组 `[]`

### 2.4 `<script setup>` 内部顺序

```
1. Vue 核心导入          import { ref, computed, watch, onMounted } from 'vue'
2. 路由导入              import { useRouter } from 'vue-router'
3. i18n 导入             import { useI18n } from 'vue-i18n'
4. Tauri API 导入        import { invoke } from '@tauri-apps/api/core'
5. 类型导入              import type { PresetData } from '@/types/preset'
6. Store 导入            import { useTaskStore } from '@/store/taskStore'
7. Hooks 导入            import { useUpdateChecker } from '@/hooks/useUpdateChecker'
8. 工具导入              import { FFmpegCommandBuilder } from '@/utils/commandBuilder'
9. 全局组件导入          import ConfirmDialog from '@/components/Dialogs/ConfirmDialog.vue'
10. 本地组件导入         import AudioSettings from './components/AudioSettings.vue'

11. const { t } = useI18n()
12. Store 初始化         const taskStore = useTaskStore()
13. defineProps / defineEmits
14. ref / computed       所有响应式状态
15. watch
16. 普通函数
17. onMounted / onUnmounted
```

### 2.5 模板规范

- 组件标签使用 `PascalCase`：`<SwitchToggle />`
- HTML 元素使用 `kebab-case`：`<div class="form-group">`
- CSS 类名使用 `kebab-case`：`.file-list`, `.form-group`
- 根元素类名：`.page-name` 或 `.component-name`

## 三、样式规范

### 3.1 CSS 变量

所有颜色必须使用 CSS 变量 + 回退值：

```css
.element {
  background: var(--bg-color1, #181818);
  color: var(--text-color1, #c0c0c0);
  border: 1px solid var(--border-color1, #444);
}
```

**可用变量：**

| 变量 | 用途 |
|------|------|
| `--bg-color1` ~ `--bg-color5` | 背景色（数字越大越亮） |
| `--text-color1`, `--text-color2` | 文字色 |
| `--border-color1` | 边框色 |
| `--info-color` | 信息/主色调（蓝） |
| `--success-color` | 成功（绿） |
| `--error-color` | 错误（红） |
| `--warning-color` | 警告（黄） |
| `--hover-bg` | 悬停背景 |

### 3.2 禁止事项

- **禁止**硬编码颜色值（`#fff`, `rgb(...)` 等）
- **禁止**使用 `!important`
- **禁止**非 scoped 样式

## 四、状态管理规范

### 4.1 Store

新 Store **必须**使用 Setup 函数风格：

```typescript
export const useXxxStore = defineStore("xxx", () => {
  const data = ref<Type[]>([]);
  const count = computed(() => data.value.length);
  function add(item: Type) { data.value.push(item); }
  return { data, count, add };
});
```

### 4.2 Hooks

- 文件名 `usePascalCase.ts`
- 返回对象 `{ state, action1, action2 }`

```typescript
export function useXxx() {
  const state = ref<StateType>({ ... });
  async function doSomething() { ... }
  return { state, doSomething };
}
```

## 五、i18n 规范

### 5.1 Key 命名

```
common.{key}           通用 UI 文案
nav.{key}              导航标签
page.{pageName}.{key}  页面专属文案
task.status.{key}      任务状态
message.{key}          提示消息
about.{key}            关于页
dialog.{name}.{key}    对话框
```

### 5.2 使用

```vue
{{ t('page.home.title') }}
{{ t('about.updateAvailable', { version: '1.0.0' }) }}
```

- **所有**用户可见文本必须通过 `t()` 获取
- 新增 key 必须同步添加到 `zh-CN.ts` 和 `en-US.ts`

## 六、类型定义

- 跨模块复用的类型放在 `src/types/` 下
- 仅单个组件/文件使用的类型内联定义
- 接口使用 `interface`，不强制 `I` 前缀
- 枚举使用 `type` 联合或 `const` 对象，避免 TypeScript `enum`

---

## 七、导入规范

- 内部模块使用 `@/` 别名
- 同级/子级组件使用相对路径 `./`
- 类型导入使用 `import type { ... }`
- 全局组件导入路径带 `.vue` 后缀，本地不带也可以

---

## 八、路由规范

- 路径：`kebab-case`（`/independent-panel`）
- 路由名：`PascalCase`（`IndependentPanel`）
- 页面组件：懒加载 `() => import('@/pages/.../index.vue')`

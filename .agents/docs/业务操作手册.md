## 1. 软件概述

FFmpeg Desktop 是一款基于 Tauri 2.x + Vue 3 构建的 FFmpeg 跨平台图形化界面，将 FFmpeg 命令行工具的强大功能转化为直观的可视化操作。

**支持平台**: Windows / macOS / Linux

**核心能力**:
- **视频转码**: 支持 H.264/H.265/AV1/VP9/ProRes 等 38 种编码器变体，涵盖 NVIDIA/Intel/AMD/Apple 全系硬件加速
- **音频处理**: 22 种音频编码器，支持响度归一化（EBU R128）
- **视频滤镜**: 反交错、降噪、锐化、裁剪、缩放、插帧、超分辨率、动态模糊、色彩管理、字幕烧录
- **批量任务**: 多任务队列并发，实时进度监控，暂停/恢复/停止
- **画质评测**: VMAF / SSIM / PSNR 多指标对比
- **媒体分析**: ffprobe 全量媒体信息查看
- **预设管理**: 完整的编码预设保存/导入/导出

## 2. 安装与启动

### 2.1 前置依赖

- **FFmpeg**: 自行安装或通过应用内一键安装
- **WebKit2GTK**（Linux）: `sudo apt install libwebkit2gtk-4.1-dev`

### 2.2 启动应用

```bash
pnpm install
pnpm tauri dev    # 开发模式
pnpm tauri build  # 生产打包
```

### 2.3 FFmpeg 配置

首次使用需要确保 FFmpeg 可用：

1. 进入 **设置 → FFmpeg** 页
2. 点击「测试 FFmpeg」检测是否已安装
3. 未安装时可点击「一键安装」自动安装
4. 或手动指定 FFmpeg/FFprobe 可执行文件路径

## 3. 快速入门

标准转码流程只需 4 步：

| 步骤 | 操作 | 对应页面 |
|------|------|----------|
| **1. 准备文件** | 拖拽或添加视频文件 | `/prepare` |
| **2. 设定参数** | 选择编码器、分辨率、画质等 | `/params` |
| **3. 加入队列** | 点击「添加到队列」生成任务 | `/prepare → /queue` |
| **4. 开始转码** | 点击「启动全部」执行任务 | `/queue` |

## 4. 页面功能详解

### 4.1 首页

路由: `/home`

展示欢迎信息和快捷入口卡片，点击可跳转到对应功能页。底部提供键盘快捷键速查表。

### 4.2 准备文件

路由: `/prepare`

#### 添加文件
- **拖拽添加**: 从文件管理器拖入视频文件
- **按钮添加**: 点击「添加文件」选择单个文件，或「添加文件夹」批量导入
- **支持格式**: `.mp4`, `.mkv`, `.avi`, `.mov`, `.flv`, `.webm`, `.wmv`, `.m4v`

#### 文件列表操作
- `Ctrl + 点击`: 多选
- `Shift + 点击`: 范围选择
- 「排序」按钮按文件名排序
- 「移除」按钮删除选中项

#### 加入编码队列
选中文件后点击「添加到队列」，系统将使用当前预设参数为每个文件生成 FFmpeg 命令行并跳转到编码队列页。

### 4.3 参数面板

路由: `/params`

这是应用核心页面，包含 13 个配置子标签。

#### 总览
显示当前参数生成的 FFmpeg 命令行预览和关键参数摘要。可复制命令行直接使用。

#### 输出设置

| 参数 | 说明 |
|------|------|
| 输出容器 | MP4 / MKV / WebM / AVI / MOV / FLV / TS / M2TS / WMV / 3GP / OGG |
| 不使用输出文件参数 | 跳过 `-o` 参数 |
| 输出目录 | 留空则与源文件同目录 |
| 自动命名 | 支持仅后缀/前缀/替换文件名/完全自定义 |
| 时间戳保留 | 保留创建/修改/访问时间 |

> **兼容性提示**: 选择容器和编码器后，若不兼容会显示黄色警告条（如 WebM 容器不支持 H.265）。

#### 视频编码器

| 类别 | 编码器选项 |
|------|-----------|
| H.264/AVC | libx264, NVENC, QSV, AMF, VAAPI, VideoToolbox, Vulkan |
| H.265/HEVC | libx265, libkvazaar, NVENC, QSV, AMF, VAAPI, VideoToolbox, Vulkan |
| AV1 | libaom-av1, libsvtav1, librav1e, NVENC, QSV, AMF, VAAPI, Vulkan, D3D12 |
| VP9 | libvpx-vp9, libsvt_vp9, QSV, VAAPI |
| VP8 | libvpx, VAAPI |
| MPEG-4/2 | mpeg4, mpeg2video |
| ProRes | prores_ks, prores_aw |
| Theora | libtheora |
| 特殊 | copy（流拷贝）, disable（无视频）, custom（自定义） |

每个编码器有独立的 preset（速度预设）、profile（编码档次）、tune（优化调优）选项。

#### 视频帧设置

- **分辨率**: 预设（Full HD/HD/4K/2K/480p/360p）或自定义宽高
- **裁剪**: 输入 crop 滤镜参数或打开交互式裁剪窗口
- **帧率**: 预设值或保持原样
- **高级功能**:
  - 插帧参数 — 视频补帧（motion interpolation）
  - 超分参数 — AI 超分辨率（libplacebo）
  - 动态模糊 — 帧混合（tblend）

#### 画质控制

| 模式 | 说明 | 典型值 |
|------|------|--------|
| CRF | 恒定质量（H.264/H.265） | 18-28（越小越好） |
| CQP | 恒定量化（硬件编码器） | 18-30 |
| VBR | 可变码率 | 设置目标/最小/最大码率 |
| CBR | 恒定码率 | 设置固定码率 |

#### 音频设置

- **编码器**: AAC, MP3, Opus, FLAC, AC3, DTS, PCM 等 22 种
- **码率**: 64-320 kbps 可选
- **采样率**: 8000-96000 Hz
- **声道**: 单声道/立体声/5.1/7.1
- **响度归一化**: EBU R128 标准，设置目标响度、动态范围、峰值电平

#### 色彩管理

- **基础调整**: 亮度、对比度、饱和度、伽马
- **像素格式**: YUV 4:2:0/4:2:2/4:4:4 8/10/12-bit, RGB
- **色彩空间**: BT.709 / BT.2020 / DCI-P3
- **HDR 转 SDR**: 10 种色调映射算法

#### 滤镜

| 滤镜 | 选项 |
|------|------|
| 反交错 | YADIF / BWDIF，各 2 种模式 |
| 降噪 | HQDN3D（快速）/ NLMeans（高质量）/ ATADenoise（时域）/ BM3D |
| 锐化 | Unsharp（亮度分量，可调半径和强度） |
| 旋转 | 不旋转/90°/180°/270°，水平/垂直翻转 |

点击「高级功能」按钮可进入插帧、超分、动态模糊的详细配置对话框。

#### 裁剪

- **裁剪方法**: 时间区间（入点/出点）、持续时长
- **入点/出点**: 支持 `HH:MM:SS` 或秒数
- **Seek Backward**: 提高裁剪精度

#### 字幕烧录

- **来源**: 无/外挂字幕文件（SRT/ASS/SSA/SUB）/内嵌字幕流
- **完整 ASS 样式**: 字体、字号、粗细、斜体、下划线、删除线、主色/副色/轮廓色/背景色+透明度、边框样式、轮廓宽度、阴影距离、对齐位置、边距、字符间距、行距、自定义样式

#### 流控制

- **视频/音频流映射**: 指定流索引
- **保留其他流**: 控制是否保留未映射的流
- **字幕操作**: 烧录/混流/丢弃
- **自动混流**: SRT/ASS/SSA 自动识别混流
- **元数据处理**: 保留或删除 metadata/chapters/attachments

#### 自定义参数

11 个自定义参数槽位，可插入任意 FFmpeg 参数:

| 插槽 | 插入位置 |
|------|---------|
| startParams | 命令开头 |
| videoFilter | `-vf` 视频滤镜 |
| audioFilter | `-af` 音频滤镜 |
| filterComplex | `-filter_complex` 复杂滤镜 |
| videoParams | `-c:v` 视频编码参数后 |
| audioParams | `-c:a` 音频编码参数后 |
| beforeOutputParams | 输出文件前 |
| afterOutputParams | 输出文件后 |
| endParams | 命令结尾（如 `-y`） |
| fullCustom | **完全自定义**命令（忽略所有其他设置），支持 `$INPUT`/`$OUTPUT` 占位符 |

#### 图像编码

13 种图像编码器: PNG, APNG(动态), JPEG, WebP, GIF, BMP, TIFF, DPX, EXR, JPEG 2000, JPEG LS, SVT JPEG XS, HDR(Radiance)

### 4.4 编码队列

路由: `/queue`（默认页）

#### 任务生命周期

```
等待中 → 处理中 → 完成
              ↓
            暂停 ⇄ 处理中
              ↓
            停止 / 错误
```

#### 实时进度

每个任务显示:
- 当前帧 / FPS / 时间 / 码率 / 速度 / 文件大小 / 百分比 / 预估剩余时间 / 预估输出大小

#### 操作按钮

| 操作 | 说明 |
|------|------|
| 启动全部 | 启动所有等待中的任务 |
| 暂停全部 | 暂停所有运行中的任务（Unix: SIGSTOP / Windows: NtSuspendProcess） |
| 停止全部 | 停止所有运行中的任务 |
| 移除选中 | 删除选中的任务 |
| 重置选中 | 将已完成/错误任务重置为等待状态 |
| 上移/下移 | F3/F4 调整任务优先级 |
| 全选 | Ctrl+A |
| 反选 | Alt+A |

#### 输出日志

底部可展开输出面板，查看每个任务的 FFmpeg stderr 日志，支持「仅错误」过滤和自动滚动。

#### stdin 交互

运行中的任务可通过底部输入框发送指令（如按 `q` 键安全退出 ffmpeg）。

### 4.5 媒体信息

路由: `/mediainfo`

使用 ffprobe 分析媒体文件：

- **基本信息**: 文件名、容器格式、时长、文件大小、总码率
- **视频流**: 编码器、分辨率、帧率、像素格式、宽高比
- **音频流**: 编码器、采样率、声道数、码率、语言
- **字幕流**: 编码格式、语言
- **元数据**: 所有 format 级别的 metadata 键值对

支持最近文件历史（最多 10 个）。

### 4.6 播放器

路由: `/player`

使用 ffplay 后端 + HTML5 视频播放器预览视频文件。

- 支持格式: `.mp4`, `.mkv`, `.avi`, `.mov`, `.webm`, `.flv`, `.wmv`
- 支持拖拽打开文件
- 仅用于快速预览和调试，非完整播放器

### 4.7 画质评测

路由: `/quality`

对比原始视频与压缩后视频的画质差异：

#### 评测指标

| 指标 | 说明 | 方向 |
|------|------|------|
| VMAF | Netflix 视频多方法评估融合 | 越高越好（0-100） |
| SSIM | 结构相似性 | 越高越好（0-1） |
| PSNR | 峰值信噪比 | 越高越好 |

#### 操作步骤

1. 选择**参考视频**（原始未压缩）和**失真视频**（压缩后）
2. 勾选评测指标（可多选，并行评估）
3. VMAF 可选择模型（默认/4K/Negative）
4. 可选导出 JSON/XML/CSV 报告
5. 点击「开始评测」
6. 查看实时进度条和日志
7. 完成后显示各指标均值/最小值/最大值

> 多指标同时评估时，系统自动通过 `split` 滤镜并行计算，不影响性能。

### 4.8 混流工具

路由: `/mux`

将多个媒体文件的音视频流重新组合到一个容器中。

- 添加多个源文件
- 为每个文件选择要提取的流（视频/音频/字幕索引）
- 可选择使用某个文件的 chapters/metadata
- 使用 `-map` + `-c copy` 无损混流
- 通过 `StreamSelectorDialog` 可视化选择流

---

### 4.9 合并工具

路由: `/merge`

将多个视频文件按顺序拼接为一个文件。

- 添加文件并拖拽排序
- 使用 FFmpeg concat demuxer
- `-c copy` 无损合并（要求所有文件编码格式一致）
- 最少需要 2 个文件

## 7. 常见工作流

### 7.1 视频压缩（H.265）

1. `/prepare` → 添加视频文件
2. `/params` → 视频编码器 → H.265 → libx265 → preset: medium
3. 画质控制 → CRF → 值设为 23
4. 音频 → AAC → 128 kbps
5. 加入队列 → 启动全部

### 7.2 硬件加速编码（NVIDIA）

1. `/params` → 视频编码器 → H.265 → hevc_nvenc
2. 画质控制 → CQP → 值设为 23
3. 加入队列 → 启动全部

### 7.3 视频无损裁剪

1. `/params` → 裁剪标签 → 设置入点/出点
2. 视频编码器 → copy（流拷贝）
3. 音频 → copy
4. 加入队列 → 启动全部

### 7.4 画质对比评测

1. 先用两个不同参数转码同一视频，得到原始版和压缩版
2. `/quality` → 选择参考文件和失真文件
3. 勾选 VMAF + SSIM + PSNR
4. 开始评测 → 查看结果

### 7.5 字幕烧录

1. `/params` → 字幕标签 → 选择外挂字幕文件或内嵌流
2. 设置样式（字体/字号/颜色/位置）
3. 加入队列 → 启动全部

### 7.6 批量格式转换

1. `/prepare` → 批量添加文件夹
2. `/params` → 输出容器选目标格式
3. 视频编码器 → copy（如需重新编码则选编码器）
4. 全选文件 → 添加队列 → 启动全部

### 7.7 视频补帧（24fps → 60fps）

1. `/params` → 视频帧设置 → 高级功能 → 插帧参数
2. 目标帧率: 60 → 模式: 运动补偿 → ME 模式: 双向
3. 视频编码器选择 → 加入队列

## 8. 故障排除

| 问题 | 解决方案 |
|------|---------|
| 找不到 FFmpeg | 设置 → FFmpeg → 测试/安装/手动指定路径 |
| 硬件编码不可用 | 确认显卡驱动已安装并支持对应编码器（如 `nvidia-smi` 检查） |
| 任务一直等待 | 检查最大并发任务数设置，确保不超出限制 |
| 字幕不显示 | 检查字体文件路径、字幕文件路径和格式 |
| 输出文件过大 | 提高 CRF 值（如 23→28）或降低码率 |
| 合并失败 | 确认所有源文件编码格式相同，或改用重新编码方式 |
| 画质评测无进度 | 确保参考和失真视频分辨率一致 |
| 容器兼容性警告 | 切换为兼容的容器格式（如 WebM → MKV） |
